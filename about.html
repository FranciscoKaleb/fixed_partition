Single Processor Fixed Partition Simulator

Overview

This simulator demonstrates memory management using fixed partitioning in a single processor environment. It combines CPU scheduling algorithms with memory allocation strategies to visualize how processes are allocated to memory partitions and executed.

CPU Scheduling Algorithms

  • FCFS (First Come First Serve): Processes are executed in the order they arrive.
  • SJF (Shortest Job First): The process with the shortest burst time is executed first.
  • NPP (Non-Preemptive Priority): Processes are executed based on priority (lower number = higher priority).

Memory Allocation Strategies

  • First Fit: Allocates the first partition that is large enough.
  • Next Fit: Similar to First Fit but starts searching from the last allocated partition.
  • Best Fit: Allocates the smallest partition that is large enough.
  • Worst Fit: Allocates the largest available partition.

Key Metrics

  • IF (Internal Fragmentation): Wasted space within allocated partitions.
  • EF (External Fragmentation): Free partitions too small for waiting processes.
  • MU (Memory Utilization): Percentage of memory being used effectively.

How to Use

  1. Select a CPU scheduling algorithm and memory allocation strategy.
  2. Add processes with arrival time, burst time, and memory requirements.
  3. Add memory partitions of various sizes.
  4. Click "Finalize Data" to start the simulation.
  5. Use ">>>" to step through or "Proceed to Finish" to complete the simulation.
